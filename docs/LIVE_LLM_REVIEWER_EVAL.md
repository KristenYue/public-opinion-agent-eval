# 真实 LLM Reviewer 小规模评估

评估日期：2026-07-11
Provider / model：DeepSeek / `deepseek-v4-flash`

## 结论

真实 provider 调用与结构化输出链路已经跑通。4 个批次全部一次成功，12 个样本全部返回合法结构化结果，无缺失、重复或额外 sample ID。

这轮实验没有证明 LLM reviewer 提升了总体准确率：LLM 在 12 条样本上的准确率为 58.3%（7/12），与 XGBoost 基线的 58.3%（7/12）持平。它纠正了 5 个基线错误中的 2 个，同时把 2 个原本正确的结果改错，净提升为 0。

## 逐类观察

- 有效纠错：将“哈哈哈”“哇哇哇”从 Neutral 修正为 Positive。
- harmful overrides：将“笑死了”“人才”从正确的 Neutral 改为 Positive，说明短文本反讽和语境依赖仍是主要风险。
- 未纠正：对“都睡不着了”“是啊 关晓彤会心疼我的”等语境型样本继续跟随了错误的 Positive 基线。
- 无意义内容：`33`、`22`、`~` 均稳定返回 Unscorable。

## 可对外表述边界

可以表述：完成 DeepSeek 真实 provider 的小规模结构化 Reviewer 评估；4/4 批次成功、12/12 覆盖，记录准确率、纠错率、延迟与 token usage。

不应表述：LLM reviewer 已提升端到端准确率；12 条样本足以代表总体效果；本次结果等同于生产压测或 provider SLA。

## 下一步

1. 扩展为按 selection reason 分层的 60–100 条评估。
2. 对“短文本 + 反讽/语境”增加 abstain 规则，避免覆盖正确基线。
3. 比较“直接覆盖”与“仅在高置信且有证据时覆盖”两种策略。
4. 固定 provider/model 版本与价格快照，再报告可复现成本。

## 探索性安全覆盖策略

在同一批 12 条响应上做离线策略回放：无条件采用 reviewer 的 5 次标签变更会纠正 2 条、伤害 2 条，最终仍为 7/12；只允许 `High confidence + model_disagreement` 覆盖时，仅发生 1 次覆盖并正确纠错，结果为 8/12，且 harmful override 为 0。

这是在同一批样本上的事后探索，存在选择偏差，不能作为独立提升证据。它的工程意义是形成下一轮预注册策略：在新增分层样本上固定规则验证，而不是继续根据结果调阈值。

该门禁随后已进入 Agent 运行时：Reviewer 原始建议始终保留用于审计，只有满足规则的变更才写入 `final_label`；每条结果同时记录 `applied` 和 `decision_reason`，工具轨迹汇总采用次数。独立样本验证完成前，运行时门禁仍应视为保守实验策略。
